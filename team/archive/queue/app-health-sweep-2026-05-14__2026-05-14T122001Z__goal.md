# Goal - eliminate bugs, unused code, and avoidable web performance cost

## Goal

Ship a narrow, independently reviewable health sweep that reduces xvision web
bugs, dead code, and avoidable performance cost without disrupting active board
tracks or deployment guardrails.

## Scope

1. Work only in `.worktrees/app-health-sweep-2026-05-14` on branch
   `app-health-sweep-2026-05-14`.
2. Start from current `origin/main`.
3. Prefer frontend and dashboard static fixes that can be verified on the
   deploy host.
4. Do not run Cargo, Docker builds, or deploy commands on this host.
5. Avoid broad rewrites and avoid touching board-owned work unless the bug is
   already merged to `main`.
6. Do not overlap unmerged PR work; if a fix touches broad ChatRail, chart, or
   eval/provider contracts, keep it minimal and document why it is safe.

## Loop

Repeat until no more safe, verifiable fixes remain in this slice:

1. Run an allowed verification or analysis command.
2. Inspect the concrete failure, warning, dead code, or performance hotspot.
3. Make the smallest code change that removes it.
4. Re-run the command that exposed it plus any adjacent focused checks.
5. Update `team/status/app-health-sweep-2026-05-14.md`.

## Definition of done

The goal is complete when:

1. At least one concrete bug, unused-code issue, or performance issue is fixed.
2. No additional safe fixes are visible from the allowed local checks without
   crossing into active board tracks.
3. Verification evidence is recorded in status.
4. A PR is opened, or a clear blocker is documented if a PR cannot be opened.
