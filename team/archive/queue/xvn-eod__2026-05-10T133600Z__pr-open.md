
per-strategy summary + audit activity + errors in the window.

## Scope adjustment from the plan

The plan's SQL targeted post-v1.1 tables (`positions`, `decisions`,
`pending_reservations`, `global_state`) that don't exist in v1. The v1
cut queries the actual session-time tables (`api_audit`, `eval_runs`,
`eval_decisions`) and renders deferred-section placeholders so the
report layout stays stable when Plan 2c + the wallet plan ship.
