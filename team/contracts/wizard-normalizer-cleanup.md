# Scope

Follow-up to `scenario-form-date-range-picker` (PR #323) per the
calendar-picker intake at `team/intake/2026-05-19-calendar-picker.md`.

The UI half of the structural fix landed first: operator-facing
free-form date and calendar inputs were replaced with a typed picker
and a canonical-set select. The remaining half is the wizard
tool-call path:

1. Tighten the `create_scenario` tool's JSON Schema so the model is
   told the canonical values up front. `enum` constraints,
   `oneOf` for the `calendar` shape, RFC3339 `format` on time
   strings, explicit object shapes for `time_window` / `capital` /
   `replay_mode` / `venue` / `data_source`.
2. Delete the Qwen-specific repair shims in
   `normalize_create_scenario_input` whose only purpose was to
   coerce a specific model's invented variant names back into a
   shape serde would accept. The calendar tag-wrap (`{"type": "Custom"}`)
   unwrap is the canonical example.
3. Keep the model-agnostic safety nets: `coerce_to_one_of` for
   enum strings, default-fill for missing required fields,
   case-insensitive lowercase aliasing.

This is the second half of the operator's 2026-05-19 frustration:
"if it's with a specific model it needs to be fixed prompt side or
systematically for all models". The picker change moved the operator
out of the bad-shape pipeline; this contract closes the model-facing
side of the same pipeline.

# Out of scope

- The engine-side `CalendarRef` enum or `ScenarioSource` enum.
  Those are upstream; if they need to change, file a separate
  contract.
- The frontend picker (already in `scenario-form-date-range-picker`
  / #323).
- Removing `coerce_to_one_of` itself. The helper is model-agnostic
  and stays as the last-line defense.
- Adding new tool-call schemas for tools beyond `create_scenario`.
  Tightening `create_strategy`, `update_slot`, etc. is a future
  cleanup that has not been asked for.
- Rewriting `infer_time_window` / `infer_asset_symbol` / the regime-
  preset alias tables. Those are general resilience patterns, not
  model-specific.
