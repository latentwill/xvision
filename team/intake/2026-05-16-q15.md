# Intake — 2026-05-16 — QA15

Operator QA report on the build at `b73b478` (after the conductor model
rolled out and after the QA10 wave merged). Six raw items; split into six
tracks under the `q15` wave.

## Raw items

1. **Granularity menu does not pop down in scenario.** Scenario create/edit
   form: the granularity (timeframe) select does not open. Likely a Radix /
   shadcn dropdown regression after recent scenario authoring work.
2. **Max tokens should match model (fetch) (agent window).** The agent
   configuration window exposes max_tokens as a manual field that does not
   default from the selected model. Agents end up running with the wrong
   ceiling and either truncate or under-fill.
3. **Make a "Retry" eval button for failures.** Failed eval runs need a Retry
   action on the run-detail page that re-queues an identical run.
4. **Scenario should pull contextual bars for bar 1 decisions, not in a
   vacuum.** If a strategy uses a 30-bar EMA and the test window is 30
   candles, decisions cannot fire until the last bar. Backtest must pre-load
   contextual/warmup bars before the scenario window so indicators are
   primed at decision t=0.
5. **Eval error — MaxTokens before any text was emitted.**
   ```
   run 01KRQGWDZ6SGQG0QTSW6FYSR33 decision 3: trader_output[truncated]:
   trader output truncated at MaxTokens before any text was emitted
   (stop_reason=MaxTokens, input_tokens=422, output_tokens=1000,
   raw_excerpt="<empty>")
   ```
   Symptom of item 2 plus a missing fallback. With reasoning/thinking
   models the entire 1000-token budget can be consumed by hidden reasoning
   before any visible text emerges. Reported run also shows three CLOSE-flat
   decisions citing "no EMA cross evident from single bar" — same root
   cause as item 4 (no warmup bars).
6. **Make all objects (eval, strategy, scenario, agent) have JSON output.**
   For eval the JSON must be **full** so it can be fed back in for QA
   round-trip. Other three need stable per-object JSON for CLI scripting.
   (The qa8 CLI JSON contract work was closed unmerged on purpose — this
   re-opens the requirement with explicit scope per object.)
