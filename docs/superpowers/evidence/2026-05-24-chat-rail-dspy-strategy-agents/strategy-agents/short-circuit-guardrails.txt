ShortCircuit enum — one distinct variant per spec §4.2 class
------------------------------------------------------------
Each has a stable .code(), a non-empty .remediation(), and a
.to_typed_error() mapping into a UnifiedPayload::Error* event payload
(observability crate). .to_cli_json() renders machine-readable JSON.

  code                                  -> typed-error payload (event_kind)
  --------------------------------------------------------------------------
  missing_tool                          -> ErrorMissingTool      (error_missing_tool)
  disabled_tool                         -> ErrorMissingTool      (error_missing_tool)
  provider_unavailable                  -> ErrorProviderUnavailable (error_provider_unavailable)
  missing_prompt                        -> ErrorMissingCapability (error_missing_capability)
  invalid_output_schema                 -> ErrorInvalidSchema    (error_invalid_schema)
  empty_demo_set                        -> ErrorPolicyDenied     (error_policy_denied)
  stale_optimized_prompt                -> ErrorInvalidSchema    (error_invalid_schema)
  filter_signal_requested_but_absent    -> ErrorMissingCapability (error_missing_capability)
  strategy_references_unattached_slot   -> ErrorMissingCapability (error_missing_capability)
  dashboard_artifact_without_persisted_row -> ErrorPersistenceFailed (error_persistence_failed)

(missing_tool and disabled_tool both map to ErrorMissingTool but carry
DISTINCT codes, asserted by the disabled_tool regression test.)

Conductor wire-in sites (NOT done here — pure detection only)
-------------------------------------------------------------
  missing_tool        -> eval-launch preflight / dispatch_capability tool-bind
                         (alongside diagnostics::required_tools_for)
  disabled_tool       -> server-side tool-policy check (Phase 2.3), when a
                         required tool resolves to Denied/disabled
  provider_unavailable-> eval-launch preflight, after resolving each slot's
                         provider vs the enabled-provider list (settings::providers)
  missing_prompt      -> eval-launch preflight (hard short-circuit at dispatch)
  invalid_output_schema-> agent::execute schema-recovery, after repair attempts
                         exhausted (instead of degrading to a noop decision)
  empty_demo_set      -> optimization launch path, before kicking off optimizer
  stale_optimized_prompt-> apply-optimized-prompt step (swap-to-child / mint),
                         before writing the prompt onto the slot
  filter_signal_requested_but_absent-> agent::pipeline edge resolution, where a
                         downstream position reads filter_signals[producer_role]
  strategy_references_unattached_slot-> eval-launch preflight, after agent
                         resolution (mirrors diagnostics no-slot blockers)
  dashboard_artifact_without_persisted_row-> dashboard mutation handlers
                         (strategy/agent/scenario create+edit+delete) after persist

Forbidden files NOT touched: chat_rail.rs, wizard_loop.rs, optimization
route, diagnostics module internals, mint module.
