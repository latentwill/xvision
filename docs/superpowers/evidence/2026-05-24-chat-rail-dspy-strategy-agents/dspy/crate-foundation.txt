dspy-rs 0.7.3 API surprises vs the Phase 3.1 spike
================================================================
1. The dsrs_macros::Signature attribute macro emits 'anyhow::Result' in its generated
   MetaSignature impl, so 'anyhow' must be a DIRECT dependency of any crate using
   #[Signature]. The spike did not call this out. Added anyhow = { workspace = true }.
2. Box<dyn MetaSignature> is not Debug, so Result<BoxedSignature, _>::expect_err does not
   compile; tests match on the Err arm instead.
3. DummyLM::call(example, chat, prediction) returns the scripted 'prediction' verbatim
   (no schema parsing) — exactly the deterministic behavior wanted for CI. Output text is
   read via LMResponse.output.content().
4. All signature fields kept primitive (String/f64/bool) so the macro emits NO
   schemars::schema_for! calls => no schemars dependency needed in xvision-dspy.
