$ xvn optimize --help
Offline DSPy prompt/demonstration optimizer (Phase 3.6): run an optimization pass over a corpus, inspect/export/import results, and accept a snapshot as a child agent. See `xvn optimize --help`

Usage: xvn optimize <COMMAND>

Commands:
  run                    Run an optimization pass over a corpus for one agent slot/capability
  inspect                Show a persisted optimization run, its candidates, and snapshots
  export-demos           Export the demos of a snapshot (or demo set) as canonical JSON
  import-demos           Import a demos JSON file into the content-addressed demo store
  accept-as-child-agent  Accept a snapshot as a child agent — records the lineage edge
  revert-accepted        Revert an accepted snapshot — clears the accept flag + lineage edge
  explain-missing-data   Explain why a corpus query produced no usable training data
  help                   Print this message or the help of the given subcommand(s)

Options:
  -h, --help  Print help

$ xvn optimize run --help
Run an optimization pass over a corpus for one agent slot/capability

Usage: xvn optimize run [OPTIONS] --agent <AGENT> --slot <SLOT> --capability <CAPABILITY> --corpus <CORPUS> --optimizer <OPTIMIZER> --metric <METRIC> --rng-seed <RNG_SEED>

Options:
      --agent <AGENT>            Agent template id being optimized (pre-mint local ULID)
      --slot <SLOT>              Slot/role name within the agent (free text)
      --capability <CAPABILITY>  Capability the slot fulfils [possible values: trader, filter, decision_grader, intern, chat_authoring]
      --corpus <CORPUS>          Corpus: a saved-query string, or a path to a corpus JSON file
      --optimizer <OPTIMIZER>    Optimizer search algorithm [possible values: mipro, gepa, copro]
      --metric <METRIC>          Objective metric name (e.g. delta_sharpe, grader_score)
      --max-rounds <MAX_ROUNDS>  Maximum optimizer rounds [default: 4]
      --rng-seed <RNG_SEED>      RNG seed for demo sampling / search order
      --dry-run                  Validate corpus + capability only; do NOT mutate the store
      --live                     Use the live provider backend (opt-in; a stub in this wave — fails with a provider error). Default is the deterministic, no-network test model
      --json                     Emit a single JSON object to stdout
      --xvn-home <XVN_HOME>      Override the XVN home (otherwise XVN_HOME or ~/.xvn)
  -h, --help                     Print help
