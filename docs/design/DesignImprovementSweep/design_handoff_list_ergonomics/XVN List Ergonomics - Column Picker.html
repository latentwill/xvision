XVN · List ergonomics

The eval list shouldn’t force an invisible horizontal scroll.

Today ListCard renders a min-w-max table inside overflow-x-auto — so with 9+ columns it never compresses and always overflows, behind a scrollbar that’s invisible until hover. Below is a working fix: a persisted column picker (mirrors your existing xvn:list:{id}:density pattern), an edge-fade scroll affordance, sticky identity + status columns, the ULID folded into the name cell, and priority auto-hide. Drag the right edge of the frame to feel it respond.

Recommendations — beyond the picker

01

Fold the ULID out of its own column

A 26-char 01J9ZK4M2A… id is the widest column and the least scanned. Drop it under the strategy name as a muted second line with copy-on-click — exactly what the mobile surfaces already do via evalRunLabels. Biggest single width reclaim.High impact · low effort

02

Edge-fade + sticky columns, not a bare scrollbar

The reason scroll feels “hidden” is there’s no signal more is there. A right-edge gradient + nudge arrow announces it; pinning identity (left) and status (right) keeps context while the middle scrolls. Far better than relying on the OS bar.High impact · low effort

03

Persisted column picker — reuse the density pattern

You already persist density at xvn:list:{id}:density. Add :columns with the same hook shape. Columns gain essential (locked) and priority metadata; the picker writes a visible-set; ListCard filters its columns array. One component, every list.Medium effort · unifies all tables

04

Priority-based responsive auto-hide

Drop min-w-max. Measure the track; if columns don’t fit, retire the lowest-priority non-essential ones automatically and surface a “+N” affordance. The picker overrides it. The table compresses gracefully instead of overflowing by default.Medium effort

05

Merge the metric columns

Sharpe · Return · Max DD are three narrow right-aligned columns competing for width. A single right-aligned “Performance” cell (return large, Sharpe/DD muted beneath) reads faster and frees ~2 columns — and ties into the focal-metric language from the detail work.Medium effort

06

Make compact density horizontal too

The density toggle only changes vertical padding today. Have compact also tighten cell px and shrink the id/meta line — so power users running 100-row pages can see more columns without scrolling at all.Low effort
