## Landing-page copy patterns that work

### 1. Ship an artifact, not knowledge

Strong pages promise a visible output:

- "working web app"
- "deployed agent"
- "four projects"
- "product idea deployed"
- "B2B agent pack ready to sell"
- "app accepting payments"

Kinamix implication: the offer should not be framed as "learn Hermes". It should be **"ship a persistent AI agent app you can operate, sell, or use daily."**

### 2. Turn setup pain into the villain

Hermes/OpenClaw setup content has obvious pain points: Docker, providers, tokens, messaging gateways, profiles, MCP, hosting, observability. Good copy converts that pain into a reason to buy:

- "running in under an hour"
- "no terminal required"
- "one-click rollback"
- "permissions checklist"
- "safe agent operation"

Kinamix implication: lead with reduced fragility: **"No more half-installed agents, mystery env vars, or dead local proxies."**

### 3. Specific stack, specific outcome

Generic "AI tools" pages are weak. Stronger pages list the stack and outcome:

- Claude Code + Figma MCP + Supabase + Vercel + GitHub → functional app from design.
- Replit → two deployed apps.
- n8n + LLMs + Retell/ElevenLabs/Pinecone/Supabase → agents/voice automations/templates.
- LangChain/LangGraph/CrewAI/MCP + Docker/AWS → deployed agentic app.

Kinamix implication: choose a small canonical stack and repeat it everywhere. Suggested stack: **Hermes + Claude Code + GitHub + Supabase/SQLite + Vercel/Railway + Telegram/Discord + MCP tools**.

### 4. Cohort and community are the premium justification

Self-paced content is cheap or free. Premium pricing is justified by:

- live sessions
- office hours
- debugging help
- cohort accountability
- capstone review
- private community
- team/company workshops

Kinamix implication: paid tiers should include **live debugging and artifact review**, because Hermes/OpenClaw users will get stuck on environment, provider, and deployment issues.

### 5. Proof beats hype

Avoid "10x" and "make money with AI" as primary claims. Stronger proof formats:

- student app URLs
- before/after workflow time saved
- deployed agent demos
- GitHub repos/templates
- enterprise/team logos
- cohort completion and shipping rates
- transparent founding cohort cap

Kinamix implication: first cohort should be optimized for **case studies**, not immediate revenue maximization.

## Course size and curriculum benchmarks

### Commodity / lead-magnet size

- 45-90 minutes.
- One clear outcome: install, connect provider, run first task.
- Best for trust-building and lead capture.

### Cohort app-builder size

- 4 weeks.
- 4-8 live sessions plus office hours.
- 3-5 hours/week expected effort.
- Output: deployed working app, preferably with auth/database/deployment.

### Agent-engineering size

- 6-8 weeks.
- 20-50 contact hours or 10-20 hours of concise video plus labs.
- Output: deployed agent app with tools, memory, monitoring, and guardrails.

### Enterprise workshop size

- 2-day to 8-week rollout.
- Needs team-specific codebase/playbook, security model, adoption metrics, and support.

Primary:

Secondary:

- agencies that want to productize agent systems

Avoid at first:

- generic "AI curious" learners

### Core promise

Not: "learn Hermes".

Use:

> In six weeks, you will deploy a working agent app that can receive tasks, use tools, remember preferences, and run on a schedule — plus a repeatable playbook for turning it into a paid service or internal operating system.

### Curriculum outline

1. **Agent app opportunity and offer design**
   - What an agent app is; where it beats chatbots and n8n-only workflows.
   - Pick one target workflow: research assistant, CRM follow-up agent, content repurposer, meeting/action agent, founder ops agent.

2. **Hermes/OpenClaw architecture in plain English**
   - Messaging layer vs long-term memory layer vs coding agent layer.
   - Profiles, skills, tools, scheduled jobs, MCP/API surfaces.
   - Safety model: secrets, permissions, rollback, human approval.

3. **Local setup without bricking your tools**
   - Install and configure provider keys safely.
   - Avoid proxy/env footguns like stale `ANTHROPIC_BASE_URL`.
   - Verify health with a minimal task.

4. **First useful agent workflow**
   - Build a single-purpose agent with a narrow job.
   - Add memory and a reusable skill.
   - Connect one messaging surface, e.g. Telegram or Discord.

5. **Tool use and MCP integrations**
   - GitHub, Google Workspace, Notion/Airtable, browser/search, files.
   - Tool permission boundaries and audit logs.

6. **Deployment and operations**
   - VPS/container deployment, logs, restarts, backups, model routing, cost caps.
   - Monitoring for silent failures and runaway tasks.

7. **Productization and monetization**
   - Turn the workflow into a paid service, internal product, or template pack.
   - Pricing models: setup fee, retainer, seat license, hosted agent, template bundle.
   - Landing page and demo script.

8. **Capstone: deployed agent app**
   - Ship a working agent app with a live demo, user story, ops checklist, and pricing hypothesis.

10. **FAQ**
   - Required subscriptions, technical prerequisites, privacy/security, expected weekly time, what happens if tools change.

## Risks and mitigations

| Risk | Why it matters | Mitigation |
| --- | --- | --- |
| Tool churn | Hermes/OpenClaw/Claude Code change quickly; static lessons decay. | Principle-first curriculum, monthly update cadence, changelog calls, version-pinned setup. |
| Too technical for non-coders | Install/deploy/OAuth/env vars will create support load. | Explicit prerequisites; founding cohort only; provide cloud/devcontainer path; live setup session. |
| Commodity free content | Free tutorials can cover install basics. | Sell debugging, deployment, safety, templates, community, and outcome review. |
| Overpromising income | AI-course market is full of low-trust income claims. | Avoid income guarantees; focus on working artifacts and realistic commercialization paths. |
| Support burden | Agents fail in weird local environments. | Standardize environment, provide troubleshooting scripts, require preflight, cap cohort size. |
| Security/privacy | Agent apps handle credentials and user data. | Secrets management, scoped permissions, human approval for dangerous actions, audit logs. |

## Final take

Kinamix Academy should not compete as another AI-tool tutorial brand. The stronger wedge is:

> **Agent apps that survive contact with reality.**

That means setup, deployment, memory, tools, monitoring, cost control, safety, and monetization — plus live help when the agent breaks. The market already has vibe-coding and AI-use courses; the open gap is the path from **AI-coded demo** to **operational agent product**.
