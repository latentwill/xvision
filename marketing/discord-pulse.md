# Discord pulse — weekly log

Running notes on what target servers are debating, what threads are picking
up traction, and what xvision-relevant questions are surfacing. The operator
maintains this. One short entry per Tier-1 server per week.

Use this file to:

- Surface conversation topics worth a Remotion video or build-in-public thread
- Track which target users are repeat-active (signal for AMA timing)
- Notice macro themes (e.g., "everyone is debating perp funding-rate
  manipulation this month") that warrant a concept-explainer video
- Document any direct mentions of xvision in target servers

## Format

```markdown
## YYYY-MM-DD — week of <date>

### Mantle Discord
- **#general:** <one-line summary of what's debated>
- **#builders:** <hackathon-project chatter, project announcements>
- **#dev-support:** <recurring technical questions xvision could answer>
- **xvision mentions:** <handle + channel + one-line>
- **Followups for me:** <thread to respond to, post to draft>

### Orderly Discord
- **#strategy-chat:** ...
- **#dev-support:** ...
- **xvision mentions:** ...
- **Followups for me:** ...

### ERC-8004 / Identity WG
- ...

### Macro themes this week
- <one-line summary across all servers>

### Action items from this pulse
- [ ] <thing to do this week based on what came up>
- [ ] ...
```

## Operator rhythm

- **Daily:** lightweight presence in Tier-1 channels; respond to direct
  questions when they arise.
- **Weekly (Fridays):** write the week's pulse entry, ~15 minutes.
- **Monthly:** review the last four pulses; surface trends; update
  `methods.md` if a pattern justifies a methods change.

## What this file is NOT

- A leaderboard of "wins" — sentiment metrics are unreliable and gameable
- A list of users to DM — DMing is in the "Do not" column of `methods.md`
- A scratchpad for unrelated notes — Discord engagement only
